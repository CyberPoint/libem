EM is the function in which the whole EM algorithm comes together.

First, you set up the initial conditions (noting that now, instead of the arguments estep and mstep pass to each other, we're now passing in the structures your caller initialized). You then call kmeans to get initial cluster centroid (mu) guesses, and then iterate between the estep and mstep until convergence. EM doesn't actually return anything - instead, you're filling the containers you created when you called the function, and when EM has "finished," you can use/print out your final approximations in the caller.

Parameters:
n number of data points
m dimensionality of data
k number of clusters
X data
sigma_matrix vector of matrix pointers generated by the caller of EM that holds the sigmas calculated
mu_matrix matrix that holds the mu approximations
Pks local copy of the matrix generated by the caller of EM that holds the Pk's calculated
Returns:
the log likelihood (density) of the data (or std::numeric_limits::infinity() on error)

ParseCSV is a function that takes in your comma delineated data and parses it according to parameters given at the command line.

This is how you first define the three crucial parameters for the Kmeans and EM approximations: m - integer representing the dimensionality of the data n - number of data points k - how many clusters you want Kmeans to find This returns 1 for success, 0 for an error or failure.
